# Deploy ke Render: Panduan Lengkap

Panduan ini memandu Anda deploy **story_vape** ke Render (Laravel + Vue Inertia + PostgreSQL).

## Apa itu Render?

- **Hosting gratis** untuk web app (spins down saat idle, tapi OK untuk testing)
- **Database PostgreSQL gratis** tanpa batas waktu
- **Infrastructure as Code** via `render.yaml` — semua konfigurasi dalam satu file
- Tidak perlu credit card untuk free tier

## Persyaratan

1. **GitHub account** (repo ini harus di GitHub — public atau private + grant Render akses)
2. **Render account** (gratis, login via GitHub)
3. **Git** sudah push ke GitHub

## Step-by-Step

### 1. Push Kode ke GitHub

```bash
git remote add origin https://github.com/YOUR_USERNAME/story_vape.git
git branch -M main
git push -u origin main
```

Atau jika sudah punya repo remote:
```bash
git push origin main
```

**Penting:** `render.yaml` harus ada di root folder repo (sudah saya buat).

---

### 2. Buat Akun & Login di Render

- Buka https://render.com
- Klik **Sign Up** → **GitHub** (login via GitHub akun Anda)
- Authorize Render untuk akses repo

---

### 3. Deploy via Blueprint (render.yaml)

1. Di dashboard Render, klik **New +** → **Blueprint** (atau cari **Blueprints**)
2. Pilih GitHub repo `story_vape` Anda
3. Render akan otomatis **membaca `render.yaml`** dan preview:
   - 1 Web Service (`story_vape`)
   - 1 PostgreSQL Database (`story_vape-db`)
4. Klik **Deploy**
5. **Tunggu ~3-5 menit** sampai build selesai

Saat build, Render akan:
- Jalankan `composer install` + `npm install` + `npm run build`
- Generate `APP_KEY` otomatis
- Buat PostgreSQL database
- Jalankan `php artisan migrate --force` (create tables)
- Start web service

---

### 4. Ambil URL & Akses Aplikasi

Saat deploy selesai, Anda akan lihat:
```
Web Service: https://story-vape.onrender.com
Database: <auto-generated-name>
```

Buka https://story-vape.onrender.com di browser.

**Catatan:** Pertama kali load mungkin slow (spins up server), tunggu ~30 detik.

## Hal-Hal yang Perlu Diketahui (Limitation Testing)

| Aspek | Behavior | Implikasi |
|-------|----------|-----------|
| **File Upload** | Files disimpan di `/storage/app`, ephemeral (hilang saat re-deploy atau restart) | Cocok untuk test behavior, bukan production. Untuk real production, gunakan S3. |
| **Database** | PostgreSQL free, persisten selamanya | ✅ Data tetap antar deploy |
| **Session/Cache** | Database-driven | ✅ Works fine |
| **Queue** | QUEUE_CONNECTION=database (sync mode) | ✅ Background jobs jalan instant (tidak async, tapi untuk testing OK) |
| **Server Activity** | Idle >15 menit → spin down (gratis tier) | Akan auto-wake saat ada request pertama. Restart ~30s. |
| **Timeouts** | No strict timeout, tapi Render recommend <30s | Generate PDF/laporan besar sebaiknya async, tapi untuk test OK. |

**Untuk testing POS Anda:**
- ✅ Dashboard, inventory, POS checkout → semuanya jalan
- ✅ Database transactions, FIFO batching → kerja normal
- ✅ Session, permissions → OK
- ⚠️ Upload file test (misal foto produk) → hilang saat re-deploy (tapi bisa reupload)
- ✅ Generate PDF laporan → jalan, tersimpan di `/tmp` (tidak persisten)

## Troubleshoot

### Build Failed

Jika deploy gagal, klik **Logs** di Render dashboard. Cari error message.

Common issues:
- **Composer/npm error:** Cek `composer.json` & `package.json` valid
- **Database migration error:** Check `database/migrations/` valid, jika ada typo di migration
- **Permission error:** Render user bukan root. Jika app butuh special folder, buat di `/tmp` atau `/mnt/data`

### Aplikasi Tidak Bisa Connect Database

Render auto-inject `RENDER_DATABASE_*` env vars dari PostgreSQL service. Jika error, cek:
- Render dashboard → Environment → verify `RENDER_DATABASE_*` vars ada dan benar
- atau, di **Settings** → **Environment** di Render UI, manual inspect nilai DB vars

### Saat Deploy, Aplikasi Load Lambat

First time load: ~30s (server spinning up). Normal untuk free tier. Reload lagi, lebih cepat.

## Cara Update Aplikasi (Setelah Deploy)

Kapan saja Anda ubah kode di GitHub dan push:

```bash
git add .
git commit -m "fix: something"
git push origin main
```

Render **auto-redeploy** dalam beberapa detik. Lihat **Deployments** tab di Render dashboard.

---

## Cara Rubah Environment Variables

Ada 2 cara:

**Option 1: Via Render Dashboard (Quick)**
1. Buka service di Render dashboard
2. **Settings** → **Environment**
3. Edit variables, save
4. Service auto-restart

**Option 2: Update render.yaml & Push (IaC)**
1. Edit `render.yaml` di repo lokal, ubah env vars
2. `git push`
3. Render auto-redeploy dengan nilai baru

## Next Steps untuk Production

Saat siap production (bukan testing):
1. **File Storage:** Ganti ke S3/Cloudinary untuk persistence
2. **Queue:** Buat dedicated queue worker service di Render
3. **Database:** Upgrade dari free PostgreSQL tier ke paid tier Render
4. **Web Server:** Ganti dari `php artisan serve` ke Laravel Octane atau Nginx+PHP-FPM (lebih optimal)
5. **SSL:** Render sudah auto-HTTPS ✅
6. **Backups:** Setup database backups di Render
